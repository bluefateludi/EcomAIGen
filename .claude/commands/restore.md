将工作区还原到最新提交的干净状态（丢弃所有未提交的修改）

执行以下步骤：

1. 运行 `git status` 查看当前工作区状态，并展示给用户
2. 运行 `git restore .` 丢弃所有已跟踪文件的修改（还原到 HEAD）
3. 询问用户是否也要清理未跟踪文件（untracked files）：
   - 如果用户确认，运行 `git clean -fd` 删除未跟踪的文件和目录
   - 如果用户不确认，跳过此步骤
4. 运行 `git status` 再次确认工作区已恢复干净状态，展示给用户

注意事项：
- `git restore .` 只恢复已跟踪文件，不会删除 untracked 文件
- `git clean -fd` 会永久删除未跟踪文件，操作不可逆，必须先询问用户确认
- 不要使用 `git reset --hard`，除非用户明确要求
